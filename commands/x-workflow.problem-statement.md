## Purpose

Problem Definition — One-sentence definition + constraints. Use when discovery is done and you need to crystallize the problem.

## Execution

1. **Definition** — Single sentence expressing the problem. Others should understand it; they can look at prior work for context.
   - **Good:** "Getting to 500 requests per second on this service increases average response time from 10ms to 500ms."
   - **Bad:** "The service response time is affected by the number of requests per second."

2. **Constraints** — List all conditions and their importance. Systems behave differently under different conditions.

## Guidelines

- You may have discovered multiple problems—define each separately; don't force one
- If multiple problems, create multiple definitions
- Proceed to `/x-workflow.research` or `/brainstorm` for Search For Solutions
