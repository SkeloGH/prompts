## Purpose

Search For Solutions — Research, modeling, prototyping. Use when the problem is defined and you need to find and validate solutions. (For brainstorming, use `/brainstorm`.)

## Execution

1. **Research** — Many problems are already solved (perhaps expressed differently). Research. If not solved, learn what worked or failed elsewhere.

2. **Modeling** — Discard ideas that don't work. Choose the best that fits constraints. Use diagrams, tests, documentation. Discard what doesn't match the problem definition.

3. **Prototyping** — A prototype is a simplified system that proves the component can be built and will work. Not a bad implementation—a validation tool.

## Guidelines

- For ideation/brainstorming, use `/brainstorm` first
- Test ideas against constraints
- Proceed to `/spec` when solution is chosen
