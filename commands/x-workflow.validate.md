## Purpose

Data Analysis — Validate that implementation is correct. Use when implementation is done and you need to verify the hypothesis and constraints.

## Execution

1. **Test the hypothesis** — Use tests from the discovery stage. Those were the problems you were trying to solve.

2. **Test constraints** — Does it work on that old browser? Does it freeze on slow network? Does the UX explain itself to a test user?

3. **Build a report** — Gather all data. Document what you achieved.

## Guidelines

- Use tests established during Problem Discovery
- Do not skip constraint testing
- Proceed to `/compound` or document outcomes for Planning For The Future
